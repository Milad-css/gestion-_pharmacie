1. Présentation Générale du Projet
1.1 Contexte et Problématique

Le secteur pharmaceutique fait face à des défis croissants en matière de gestion opérationnelle : suivi des stocks, gestion des dates d'expiration, traitement des commandes clients et tenue d'un catalogue produits à jour. Les méthodes manuelles ou les outils non dédiés génèrent des erreurs, des ruptures de stock non détectées et une expérience client dégradée.

Dans le cadre de notre projet de fin d'études en Développement Digital Web Full Stack, nous avons identifié ce besoin et proposé une solution web moderne, accessible depuis n'importe quel navigateur, permettant à une pharmacie de gérer centralement ses opérations et d'offrir un service de commande en ligne à ses clients.

1.2 Objectifs du Projet

Le projet PharmaCie vise à :

  • Développer une application web full stack complète et responsive
  • Permettre aux clients de consulter, rechercher et commander des médicaments en ligne
  • Offrir un tableau de bord d'administration pour piloter les produits, stocks et commandes
  • Implémenter un système d'authentification sécurisé avec gestion des rôles (Admin / Client)
  • Garantir la cohérence des données (transactions atomiques, contrôle de stock)
  • Appliquer les meilleures pratiques : architecture MVC, API REST, React Context, Sanctum

3.2 Architecture du Système

L'application suit une architecture client-serveur découplée. Le frontend et le backend sont deux projets indépendants qui communiquent exclusivement via une API REST JSON sécurisée.

Flux de communication

Navigateur ↔ Axios + Bearer Token ↔ Laravel API (port 8000) ↔ Eloquent ORM ↔ MySQL (port 3306)

Frontend — Organisation des fichiers clés :

FichierRôle
src/api/axios.jsInstance Axios avec baseURL et intercepteur Authorization header
src/context/AuthContext.jsxContext d'authentification — user, login, register, logout + useAuth()
src/context/CartContext.jsxContext du panier — cart, itemCount, addToCart, updateItem, removeItem + useCart()
src/components/ProtectedRoute.jsxGarde de route : ProtectedRoute (auth) et AdminRoute (admin)
src/components/Navbar.jsxNavigation adaptative selon le rôle (visiteur / client / admin)
src/App.jsxArbre de providers : AuthProvider → CartProvider → routes

3.3 Base de Données

La base de données MySQL se compose de 6 tables principales liées par des clés étrangères :

TableColonnes principalesDescription
usersid, name, email, password, role, telephone, adresseComptes utilisateurs — rôle : admin / client
categoriesid, nom, descriptionCatégories pharmaceutiques
productsid, category_id, nom, description, prix, stock, image, date_expiration, actifCatalogue de médicaments
ordersid, user_id, total, statut, adresse, telephone, notesCommandes clients — statuts : en_attente, confirmee, livree, annulee
order_itemsid, order_id, product_id, quantite, prix_unitaireLignes de chaque commande
cart_itemsid, user_id, product_id, quantitePanier temporaire par utilisateur

Relations Eloquent :

  • User hasMany Orders  |  User hasMany CartItems
  • Category hasMany Products  |  Product belongsTo Category
  • Order hasMany OrderItems  |  OrderItem belongsTo Product
  • CartItem belongsTo Product

4. Exigences Non Fonctionnelles
ExigenceDescriptionMise en œuvre
Sécurité Protection des données et des accès — Token Bearer Sanctum avec révocation à la déconnexion
— Hash bcrypt (12 rounds) pour les mots de passe
— Middleware AdminMiddleware (HTTP 403 si rôle insuffisant)
— Validation stricte de toutes les entrées Laravel
— Vérification d'appartenance des ressources (panier, commandes)
— CORS restreint aux origines autorisées
— Transaction DB atomique lors de la création d'une commande
Performance Réactivité de l'interface — Pagination côté serveur (20 produits/page)
— Recherche déclenchée à chaque frappe (debounce côté client)
— Bundler Vite avec Hot Module Replacement
— Images servies depuis le stockage public Laravel
Maintenabilité Code clair et structuré — Architecture MVC côté Laravel
— React Context API pour la gestion d'état global découplée
— Composants React fonctionnels avec hooks
— Séparation claire pages / composants / context / api
Compatibilité Accessibilité multi-navigateurs — Application SPA responsive (Tailwind CSS)
— Compatible Chrome, Firefox, Edge, Safari modernes
Disponibilité Fonctionnement en développement — Backend : localhost:8000 (php artisan serve)
— Frontend : localhost:5173 (npm run dev)
— Prêt pour déploiement cloud (Railway, Render, VPS)
